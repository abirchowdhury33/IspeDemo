Data protection & cyber threat prevention
🔐
ABAC Access Control
AAS Security Part 4 — attribute-based policies. No static passwords. Role + context + time-of-day enforced at the submodel property level.
🚫
Data Leakage Prevention
IP and pricing data compartmentalised in separate submodels. Rosemary cannot access Roche formulation data. Roche cannot access Rosemary cost structure.
👁️
Anomaly Detection
Every API call logged with actor, timestamp, IP, and data diff. Unusual access patterns (bulk export, off-hours reads) trigger security alert within 60s.
🔗
End-to-End Encryption
TLS 1.3 in transit. AES-256 at rest. AASX file packages are signed with X.509 certificates — tamper-evident, non-repudiable chain of custody.

Key technical standards & endpoints
📡
AAS Part 2 (API): GET /submodels/{id}, PUT /submodels/{id}/elements — standard IDTA REST interface, no custom integration needed.
🔐
ABAC policy engine: Policies defined as JSON — role + assetId + submodelId + operation → allow/deny. No ACL table maintenance.
🗃️
No vendor lock-in: IDTA open standard. Deploy BaSyx, Eclipse AAS4J, or any compliant runtime. Migrate between vendors without data loss.
📦
AASX offline mode: Small suppliers without API support send signed AASX packages. Same data model, zero integration effort on their end.
